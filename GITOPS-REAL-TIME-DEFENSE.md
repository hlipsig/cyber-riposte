# GitOps + Real-Time Autonomous Defense

**Concept**: AI defender can take immediate action while maintaining GitOps as source of truth  
**Key Innovation**: `flux suspend` enables emergency changes without breaking GitOps workflow  
**Status**: Design proposal for implementation across scenarios

---

## The Problem with Pure GitOps

Traditional GitOps workflow:
```
Threat Detected → Human Reviews → Commits to Git → Flux Reconciles → Defense Applied
⏱️ Timeline: Minutes to hours
```

**Too slow for:**
- Active intrusions
- DDoS attacks
- Zero-day exploits
- Data exfiltration in progress

---

## Our Solution: Suspend → Act → Commit → Review

```
Threat Detected → AI Suspends Flux → Applies Defense NOW → Commits to Git → Human Reviews → Resume Flux
⚡ Timeline: Milliseconds for defense, minutes for review
```

**Key insight**: Flux suspension allows immediate kubectl changes without GitOps fighting back, while maintaining full audit trail in Git.

---

## Core Workflow

### 1. Detection Phase
```python
# Agent detects threat
incident = {
    "incident_id": "INC-2026-06-11-001",
    "attacker_ip": "203.0.113.42",
    "threat_type": "SQL Injection",
    "confidence": 0.98,
    "severity": "HIGH"
}
```

### 2. Suspend GitOps
```bash
# Prevent Flux from reverting our emergency changes
flux suspend kustomization <app-name>

# Log the suspension
git commit --allow-empty -m "Emergency: Flux suspended for incident INC-2026-06-11-001"
git push
```

### 3. Apply Immediate Defense
```bash
# Example: Block attacker IP with NetworkPolicy
kubectl apply -f - <<YAML
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: emergency-block-203-0-113-42
  namespace: default
  labels:
    incident: INC-2026-06-11-001
    applied-by: ai-defender
    requires-review: "true"
spec:
  podSelector:
    matchLabels:
      app: vulnerable-service
  policyTypes:
  - Ingress
  ingress:
  - from:
    - ipBlock:
        cidr: 0.0.0.0/0
        except:
        - 203.0.113.42/32  # Block attacker
YAML

# ✅ Threat blocked in <1 second
```

### 4. Commit Defense to Git
```bash
# Save the NetworkPolicy
kubectl get networkpolicy emergency-block-203-0-113-42 -o yaml \
  > k8s/network-policies/emergency-block-203-0-113-42.yaml

# Commit with detailed context
git add k8s/network-policies/emergency-block-203-0-113-42.yaml
git commit -m "AI Defense: Block SQL injection from 203.0.113.42

Incident: INC-2026-06-11-001
Threat: SQL Injection attempts (98% confidence)
Action: NetworkPolicy blocking source IP
Applied: $(date -u +%Y-%m-%dT%H:%M:%SZ)
Status: ⚠️  FLUX SUSPENDED - Awaiting human review

Review Commands:
  Approve: flux resume kustomization <app-name>
  Reject:  git revert HEAD && git push && flux resume kustomization <app-name>

Evidence:
  - 47 SQLi attempts in 12 seconds
  - Signatures: UNION SELECT, OR 1=1
  - Targeted endpoints: /api/users, /api/admin

Automatically generated by AI Defender
"

git push origin main
```

### 5. Create GitHub Issue for Review
```bash
gh issue create \
  --title "🚨 AI Defense Action: Blocked 203.0.113.42" \
  --label "ai-defense,requires-review,high-severity" \
  --body "## Incident: INC-2026-06-11-001

### Threat Summary
- **Type**: SQL Injection
- **Source**: 203.0.113.42
- **Confidence**: 98%
- **Severity**: HIGH

### Action Taken
Applied NetworkPolicy blocking source IP.

### Current Status
⚠️ **Flux suspended** - App is running with emergency config.

### Review Required
**Approve** (keep defense):
\`\`\`bash
flux resume kustomization <app-name>
\`\`\`

**Reject** (remove defense):
\`\`\`bash
git revert HEAD && git push
flux resume kustomization <app-name>
\`\`\`

### Evidence
[Link to dossier](https://dossiers.../INC-2026-06-11-001)

### Timeline
- Detected: 2026-06-11T14:32:15Z
- Flux suspended: 2026-06-11T14:32:16Z
- Defense applied: 2026-06-11T14:32:17Z
- Committed to Git: 2026-06-11T14:32:20Z

/cc @security-team"
```

### 6. Human Review & Resolution

**Option A - Approve (Keep Defense)**:
```bash
# Human comments: /approve on GitHub issue
# AI executes:
flux resume kustomization <app-name>
# Flux reconciles - sees NetworkPolicy in Git - leaves it in place
# Issue auto-closes with label: approved
```

**Option B - Reject (Remove Defense)**:
```bash
# Human comments: /reject on GitHub issue
# AI executes:
git revert HEAD
git push origin main
flux resume kustomization <app-name>
# Flux reconciles - NetworkPolicy removed
# Issue auto-closes with label: rejected
```

**Option C - Modify Then Approve**:
```bash
# Human edits the NetworkPolicy in Git
# Modifies IP range, adds time-based expiry, etc.
git add k8s/network-policies/emergency-block-203-0-113-42.yaml
git commit -m "Modified AI defense: Add 24h TTL"
git push

# Then approves
flux resume kustomization <app-name>
# Flux reconciles with human-modified version
```

## Implementation Components

### 1. Flux Controller Integration

```python
# agent/flux_controller.py

import subprocess
from typing import Optional

class FluxController:
    """Manage Flux suspension/resumption for emergency changes."""
    
    def suspend(self, kustomization: str, reason: str) -> bool:
        """
        Suspend Flux kustomization to allow emergency changes.
        
        Args:
            kustomization: Name of Flux kustomization
            reason: Why we're suspending (logged to Git)
        
        Returns:
            True if suspended successfully
        """
        try:
            # Suspend the kustomization
            subprocess.run([
                "flux", "suspend", "kustomization", kustomization
            ], check=True, capture_output=True)
            
            # Commit the suspension to Git for audit trail
            subprocess.run([
                "git", "commit", "--allow-empty",
                "-m", f"Emergency: Flux suspended\n\nReason: {reason}"
            ], check=True)
            
            subprocess.run(["git", "push"], check=True)
            
            return True
        except subprocess.CalledProcessError as e:
            logger.error(f"Failed to suspend Flux: {e}")
            return False
    
    def resume(self, kustomization: str) -> bool:
        """Resume Flux reconciliation after human review."""
        try:
            subprocess.run([
                "flux", "resume", "kustomization", kustomization
            ], check=True, capture_output=True)
            
            # Log resumption
            subprocess.run([
                "git", "commit", "--allow-empty",
                "-m", "Flux resumed after human review"
            ], check=True)
            
            subprocess.run(["git", "push"], check=True)
            
            return True
        except subprocess.CalledProcessError as e:
            logger.error(f"Failed to resume Flux: {e}")
            return False
    
    def is_suspended(self, kustomization: str) -> bool:
        """Check if kustomization is currently suspended."""
        result = subprocess.run([
            "flux", "get", "kustomization", kustomization, "-o", "json"
        ], capture_output=True, text=True)
        
        import json
        data = json.loads(result.stdout)
        return data.get("spec", {}).get("suspend", False)
```

### 2. Git Committer

```python
# agent/git_committer.py

class GitCommitter:
    """Commit emergency defenses to Git with proper context."""
    
    def commit_defense(
        self,
        files: List[str],
        incident: Dict,
        action_taken: str
    ) -> str:
        """
        Commit defense action to Git with full context.
        
        Returns commit SHA for tracking.
        """
        # Add files
        for f in files:
            subprocess.run(["git", "add", f], check=True)
        
        # Build detailed commit message
        message = f"""AI Defense: {action_taken}

Incident: {incident['incident_id']}
Threat: {incident['threat_type']} ({incident['confidence']*100:.0f}% confidence)
Target: {incident['target']}
Source: {incident['attacker_ip']}
Action: {action_taken}
Applied: {datetime.utcnow().isoformat()}Z

Status: ⚠️  FLUX SUSPENDED - Awaiting human review

Review Commands:
  Approve: flux resume kustomization {incident['kustomization']}
  Reject:  git revert HEAD && git push && flux resume kustomization {incident['kustomization']}

Evidence:
{self._format_evidence(incident)}

Automatically generated by AI Defender
"""
        
        # Commit
        subprocess.run(["git", "commit", "-m", message], check=True)
        
        # Push
        subprocess.run(["git", "push", "origin", "main"], check=True)
        
        # Return commit SHA
        result = subprocess.run(
            ["git", "rev-parse", "HEAD"],
            capture_output=True, text=True
        )
        return result.stdout.strip()
```

## Safety Mechanisms

### 1. Confidence Thresholds
```python
# Only auto-suspend for high-confidence threats
if incident.confidence >= 0.95 and incident.severity == "HIGH":
    flux.suspend()
    apply_defense()
else:
    # Lower confidence → just alert, don't auto-act
    create_alert(incident)
```

### 2. Time-Based Expiry
```yaml
# Auto-expire emergency policies
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  annotations:
    expires-at: "2026-06-12T14:32:17Z"  # 24 hours
    auto-remove: "true"
```

### 3. Blast Radius Limits
```python
# Don't suspend critical infrastructure
NEVER_SUSPEND = ["core-dns", "ingress-controller", "cert-manager"]

if kustomization in NEVER_SUSPEND:
    logger.error("Cannot suspend critical infrastructure")
    alert_human_immediately()
    return False
```

### 4. Maximum Suspension Time
```python
# Auto-alert if suspended too long
if flux.suspended_duration(kust) > timedelta(hours=4):
    escalate_to_oncall(
        "Flux suspended for 4+ hours without human review"
    )
```

## Rollout Strategy

### Phase 1: Monitoring Only
- AI detects threats
- AI creates GitHub issues
- Humans manually suspend/apply/commit
- **Goal**: Build confidence, tune thresholds

### Phase 2: Automated Low-Risk Actions
- AI can auto-apply NetworkPolicies
- AI can auto-scale deployments
- Flux auto-suspended
- **Goal**: Prove the workflow works

### Phase 3: Expanded Autonomy
- AI can modify Istio rules
- AI can update container images
- AI can change quotas/limits
- **Goal**: Handle common scenarios

### Phase 4: Full Autonomous Defense
- AI handles most incidents automatically
- Humans focus on edge cases and approvals
- Continuous learning from human feedback
- **Goal**: AI as first responder
