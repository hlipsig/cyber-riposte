# Phase 8: Incident Report Templates (Template-Based)

**Status**: 🚀 Ready to implement  
**Approach**: Generate markdown templates, no GitHub API integration

---

## New Approach: Template Generation Only

Instead of integrating with GitHub API, Phase 8 will:
- ✅ Generate well-formatted incident report markdown files
- ✅ Use Jinja2 templates for flexibility
- ✅ Create evidence summaries and dossiers
- ✅ Output files that can be manually converted to issues
- ✅ No API keys or external dependencies required

**Benefits**:
- No need for separate incidents repository
- No API rate limits or authentication
- Generated reports can be used anywhere (GitHub, Jira, Slack, email)
- Easy to customize templates
- Fully testable without external services

## Implementation

### Files to Create

1. **agent/template_generator.py**:
   ```python
   """
   Template-based report generator.
   Generates incident reports without external API calls.
   """
   
   from jinja2 import Environment, FileSystemLoader
   from pathlib import Path
   from datetime import datetime
   
   class IncidentReportGenerator:
       def __init__(self, template_dir="templates"):
           self.env = Environment(loader=FileSystemLoader(template_dir))
       
       def generate_incident_report(self, incident_data, output_dir="incidents"):
           template = self.env.get_template("incident-report.md.j2")
           output = template.render(**incident_data)
           
           output_path = Path(output_dir) / f"{incident_data['incident_id']}.md"
           output_path.parent.mkdir(parents=True, exist_ok=True)
           output_path.write_text(output)
           
           return str(output_path)
       
       def generate_slack_notification(self, incident_data):
           template = self.env.get_template("slack-notification.md.j2")
           return template.render(**incident_data)
       
       def generate_executive_summary(self, summary_data):
           template = self.env.get_template("executive-summary.md.j2")
           return template.render(**summary_data)
   ```

2. **Update agent/executor.py**:
   ```python
   # After generate_postmortem()
   try:
       from agent.template_generator import IncidentReportGenerator
       
       generator = IncidentReportGenerator()
       
       # Generate incident report
       report_path = generator.generate_incident_report({
           'incident_id': incident_id,
           'attacker_ip': attacker_ip,
           'detection_signature': detection['signature'],
           'osint': osint_data,
           'actions': audit.entries,
           # ... more data
       })
       
       logger.info(f"Incident report generated: {report_path}")
       
       # Generate Slack notification content
       slack_msg = generator.generate_slack_notification({...})
       logger.info(f"Slack notification ready:\n{slack_msg}")
       
   except Exception as e:
       logger.error(f"Failed to generate incident report: {e}")
   ```

3. **templates/incident-report.md.j2** (Jinja2 template)

### Manual Usage

Once generated, users can:

1. **Create GitHub issue manually**:
   ```bash
   # Copy incident report content
   cat incidents/INC-2024-0615-0314.md
   
   # Paste into GitHub issue
   # Or use gh CLI:
   gh issue create --title "[INC-2024-0615-0314] Reconnaissance Detected" \
     --body-file incidents/INC-2024-0615-0314.md \
     --label "security,incident,high"
   ```

2. **Post to Slack**:
   ```bash
   # Copy Slack notification
   cat incidents/slack/INC-2024-0615-0314.txt
   
   # Paste into #security-incidents channel
   ```

3. **Email to security team**:
   ```bash
   # Convert markdown to HTML
   pandoc incidents/INC-2024-0615-0314.md -o report.html
   
   # Send via email client
   ```
